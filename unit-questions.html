Tivoli’s process automation engine overview

نظرة عامة على المحرك والوظائف والموديولات وكيفية تقسيم وإتاحة التطبيقات للمستخدمين.

سؤال ١
True or false: Applications are organized by modules.
💡 الإجابة الرسمية:
True. From the Go To menu, you can access a list of modules. You can then access the applications from within those modules.
📝 ملخص الدرس:
يتم تنظيم التطبيقات هيكلياً وتجميعها داخل وحدات (Modules) للوصول إليها بسهولة عبر قائمة التنقل الرئيسية.
سؤال ٢
True or false: Only an administrator with special rights can update Start Centers.
💡 الإجابة الرسمية:
False. Users can customize their own Start Centers.
📝 ملخص الدرس:
تتيح المنصة للمستخدمين العاديين تخصيص واجهات مراكز البدء الخاصة بهم (Start Centers) ولا تقتصر على المديرين.
سؤال ٣
True or false: All applications contain a List tab and other application-specific tabs.
💡 الإجابة الرسمية:
False. There are several single-page applications, including Sets, Currency Codes, Exchange Rates, and Terms and Conditions.
📝 ملخص الدرس:
هناك تطبيقات تتكون من صفحة واحدة فقط (Single-page applications) ولا تحتوي على تبويب القائمة (List tab).
سؤال ٤
True or false: You can directly access Help for an application while in that application.
💡 الإجابة الرسمية:
True. While in an application, the Help link provides direct access to help for that application.
📝 ملخص الدرس:
يوفر النظام ميزة المساعدة السياقية المرتبطة بالتطبيق المفتوح مباشرة.

Foundation data

البيانات التأسيسية: هيكل المؤسسات والمواقع والمجموعات، التسلسل الهرمي للمواقع، معيار UNSPSC والتصنيفات.

سؤال ١
Why are organizations needed in the engine?
💡 الإجابة الرسمية:
You can use organizations to have a single system with a single database. With organizations, certain entities such as security configuration can be shared, while others remain separated.
📝 ملخص الدرس:
المؤسسات (Organizations) تسمح بعزل البيانات ومشاركتها بمرونة مع استخدام سيرفر وقاعدة بيانات واحدة.

What are sets?
💡 الإجابة الرسمية:
Sets are groupings of information that can be shared across organizational boundaries.
📝 ملخص الدرس:

Tivoli’s process automation engine must have at least one organization, one company set, and one item set.

سؤال ٦
What is a location hierarchy?
💡 الإجابة الرسمية:
The hierarchical location structure is a product location hierarchy that uses a parent-child design. This structure is used to represent locations where assets operate.
📝 ملخص الدرس:

Security

الأمان والصلاحيات والمجموعات الأمنية وسجل الأشخاص مقابل حسابات المستخدمين والمصادقة والتفويض.

سؤال ١
How is a person record used?
💡 الإجابة الرسمية:
A person record contains basic information about people defined in the system.
📝 ملخص الدرس:
سجل الشخص يُخزن معلومات الموظفين الأساسية ويختلف برمجياً عن حساب المستخدم (User account).
سؤال ٢
Define authentication and authorization.
💡 الإجابة الرسمية:
Authentication uses the user ID and password to determine who is trying to access the system. Authorization determines, based on group membership, what resources an authenticated user can access.
📝 ملخص الدرس:
المصادقة (Authentication) تؤكد الهوية، والتفويض (Authorization) يحدد صلاحيات الدخول عبر المجموعات الأمنية.
سؤال ٣
What is a virtual profile?
💡 الإجابة الرسمية:
A virtual profile is created by combining two security groups.
📝 ملخص الدرس:
الملف الشخصي الافتراضي يتشكل برمجياً من دمج صلاحيات جميع المجموعات المرتبطة بالمستخدم.
سؤال ٤
How do users and security groups work together?
💡 الإجابة الرسمية:
Security groups define authorizations. The users inherit the authorizations of the groups to which they belong, which then creates their security profiles.
📝 ملخص الدرس:
يرث المستخدمون صلاحيات الوصول الخاصة بهم من المجموعات الأمنية المُلحقين بها.

سؤال ٥
What do you use E-signature and E-audit for?
💡 الإجابة الرسمية:
Electronic signatures and electronic audit records (E-Signature and E-Audit) provide an additional level of security control and auditing. Using this capability, the system administrator can perform these tasks: – Require a user to provide a specific signature when saving or changing a record or accessing a specific menu item. – Record and audit changes to records and keep copies of those changes to produce an audit trail.
📝 ملخص الدرس:
تُستخدم هذه الميزات لتوفير سجل تتبع (Audit) قوي وإلزام المستخدم بالتحقق الأمني عند الإجراءات الحساسة.

What are two of the differences between running a database configuration from the Database Configuration application and from the command line?
💡 الإجابة الرسمية:
– When running the configuration from the command line, backup tables must be deleted first. Backup tables are automatically overwritten when running the configuration from the application.
📝 ملخص الدرس:

Troubleshooting

تتبع ومعالجة الأعطال: المسجلات (Loggers)، موجهات الخرج (Appenders)، مسارات اللوجات، حاويات الويب، وسجلات JVM.

سؤال ١
What is a logger?
💡 الإجابة الرسمية:
Loggers are components in the product that prepare log statements to be written to console or log file.
📝 ملخص الدرس:
المُسجلات (Loggers) هي المكونات البرمجية التي تقوم بتحضير وتنسيق رسائل الأخطاء والأحداث.
سؤال ٢
What is an appender?
💡 الإجابة الرسمية:
Logging requests can be sent to multiple destinations. An output destination is called an appender.
📝 ملخص الدرس:
الوجهة (Appender) هي مسار الخرج النهائي (مثل ملف نصي أو شاشة) الذي تُرسل إليه رسائل المُسجلات.
سؤال ٣
How can you specify an alternate folder for logs?
💡 الإجابة الرسمية:
You can specify a folder for your log files by selecting the Set Logging Root Folder action from the Select Action menu in the Logging application.
📝 ملخص الدرس:
يمكن تغيير المسار الافتراضي لملفات السجلات (Logs) ديناميكياً عبر تطبيق إدارة السجلات.
سؤال ٤
What is a web container?
💡 الإجابة الرسمية:
A web container is a run time environment for Web applications. It processes servlets, JSP files, and other types of server-side components.
📝 ملخص الدرس:
حاوية الويب (Web container) هي البيئة المسؤولة عن تنفيذ ومعالجة ملفات واجهات المستخدم البرمجية.
سؤال ٥
How can you view the log files for the JVM associated with MXServer?
💡 الإجابة الرسمية:
The log files for the web container JVM are in the profile directory for MXServer. The logs are called SystemOut.log and SystemErr.log. Alternatively, you can view them using the WebSphere Administrative Console.
📝 ملخص الدرس:
ملفات السجلات الجوهرية (SystemOut.log و SystemErr.log) تُحفظ محلياً داخل مجلدات بيئة السيرفر أو عبر واجهة التحكم.
